`grubstation` is a Go-based agent that enables remote OS selection and booting for bare-metal servers. Integrated with [Home Assistant](https://www.home-assistant.io/) and the GRUB bootloader, it allows you to remotely select your next operating system, wake the machine via Wake-on-LAN, and boot into that OS.

Perfect for remote OS selection where you need to switch between Linux and Windows without physical access. Or if you multi-boot with a bluetooth keyboard, because GRUB doesn't load a bluetooth stack.

---

## ✨ Key Features

- 🖥️ **Remote OS Selection:** Pick your next boot option directly from Home Assistant.
- 🔍 **Auto-Discovery:** Automatically parses your GRUB configuration to find all available boot entries.
- ⚡ **Integrated WOL:** Seamlessly works with Wake-on-LAN to power on and boot into your selected OS.
- ⏻ **Power Off From Home Assistant:** Automatically configures and sets up the ability _turn off_ a machine in Home Assistant.
- 🪄 **Setup Wizard:** An interactive CLI wizard that handles configuration and installation in seconds.
- 🐧 **Cross-Platform:** Supports Linux (systemd) and Windows service management.
- 🔒 **Secure:** Uses generated Webhook IDs and Trust on First Use (TOFU) keys for secure communication with Home Assistant.

---

## 🏗️ How it Works

1. **Discovery:** The `grubstation` agent runs as a background service. It scans your GRUB configuration and pushes the list of available OS entries to Home Assistant upon system shutdown.
2. **Selection:** You select the desired OS in the [Home Assistant Integration](https://github.com/jjack/ha-grubstation).
3. **Trigger:** You trigger a "Turn On" action from Home Assistant or by manually pressing the power button.
4. **Boot Hook:** When the machine starts, a custom GRUB script (installed by the wizard) loads a network stack and then requests the next boot option from Home Assistant.
5. **Override:** Home Assistant tells GRUB which OS was selected. GRUB overrides the default entry and boots the chosen system. If nothing was selected or there's a problem with communications, everything gracefully falls back to your default boot options.

## 🚀 Quick Start

### Prerequisites
- [Home Assistant](https://www.home-assistant.io/) instance.
- The [Grubstation](https://github.com/jjack/ha-grubstation) Integration installed in Home Assistant.
- A machine using **GRUB** as the bootloader (for remote selection) and **systemd** (Linux) or **Windows** for the agent service.

### 1. Installation
Download the latest package for your architecture from the [Releases Page](https://github.com/jjack/grubstation/releases/latest).

**Debian/Ubuntu:**
```bash
sudo dpkg -i grubstation_*_amd64.deb
```

**RHEL/Fedora:**
```bash
sudo rpm -i grubstation_*_amd64.rpm
```

**Windows:**
```cmd
.\grubstation*.msi
```

### 2. Configuration Wizard
> [!TIP]
> The wizard will be able to automatically detect and configure most options except for the webhook id that's generated by the Home Assistant integration. Be sure to have that available.
Run the automated setup wizard. This will walk you through network detection, Home Assistant integration, and service installation:

**Linux**
```bash
sudo grubstation setup
```

**Windows**
> [!NOTE]
> The `.msi` installer will launch this automatically.
```cmd
"C:\Program Files\GrubStation\grubstation.exe setup"
```

## 🛠️ Usage

The `grubstation` CLI provides several commands for managing the agent and its integration.

| Command | Description |
| :--- | :--- |
| `grubstation setup` | Runs the interactive configuration and installation wizard. |
| `grubstation boot list` | Displays all boot entries discovered in your GRUB config. |
| `grubstation boot push` | Manually pushes the current boot entries to Home Assistant. |
| `grubstation service status`| Checks the status of the background agent service and daemon health. |
| `grubstation service start/stop`| Manually control the background agent service. |
| `grubstation config validate` | Validates your configuration file for errors. |
| `grubstation config init` | Generates a template `config.yaml` file. |

## ⚙️ Configuration

While the `setup` wizard is recommended, you can generate an empty config file with `grubstation config init -o config.yaml`

| OS | Default Path |
| -- | ------------ |
| Linux | /etc/grubstation/config.yaml |
| Windows | C:\ProgramData\GrubStation\config.yaml |

See [config.sample.yaml](config.sample.yaml) for a complete configuration example.

You can also use [config.shutdown.sample.yaml](config.shutdown.sample.yaml) for a shutdown-only agent.

## ⚠️ Important Notes on GRUB Networking

The remote selection feature relies on GRUB's ability to initialize your network card and perform an HTTP request. 

- **Hardware Compatibility:** UEFI and network firmware can be finicky. Some motherboards may require specific GRUB modules or your network may have long STP (Spanning Tree Protocol) delays.
- **Wait Time:** If your network takes a while to come up, you can adjust the `grub_wait_seconds` in your config (default is 2s).
- **Manual Setup:** For advanced network troubleshooting, see the [Manual Bootloader Docs](/docs/configuration/bootloader.md).
